# ISO 25010 — Software Quality Assurance

## 1. Functional Suitability

| Atributo | Implementación |
|----------|----------------|
| **Completeness** | CRUD completo para todas las entidades (usuarios, cultivos, plagas, productos, reportes, tratamientos, recomendaciones, alertas, dispositivos). |
| **Correctness** | Validación con `class-validator` DTOs en todos los endpoints. Tipado estricto TypeScript en frontend y backend. |
| **Appropriateness** | Arquitectura modular (12 módulos NestJS). Separación clara controller → service → repository. |

## 2. Performance Efficiency

| Atributo | Implementación |
|----------|----------------|
| **Time behavior** | Middleware `PerformanceMonitor` mide tiempos de respuesta (<500ms objetivo). Paginación en listas grandes. |
| **Resource utilization** | Conexiones PostgreSQL vía pool reutilizable (`pg.Pool`). MinIO para almacenamiento eficiente de imágenes. |
| **Capacity** | Migraciones Drizzle ORM optimizadas. Índices en FK y columnas de búsqueda frecuente. |

## 3. Compatibility

| Atributo | Implementación |
|----------|----------------|
| **Co-existence** | API REST stateless. Contenedores Docker independientes (PostgreSQL, MinIO, API). |
| **Interoperability** | API JSON estándar. Autenticación JWT. Mobile consume vía Axios con interceptores. |

## 4. Usability

| Atributo | Implementación |
|----------|----------------|
| **Operability** | Documentación Swagger/OpenAPI en `/api/docs`. Estados claro en reportes (PENDIENTE → COMUNIDAD → VALIDADO/RECHAZADO). |
| **User error protection** | Validación DTO con mensajes descriptivos. Filtro global de excepciones Multer. Confirmación antes de acciones destructivas. |
| **UI Aesthetics** | Diseño iOS-minimalista con Ionicons. Badges de estado con colores semánticos. Tarjetas animadas con feedback táctil. |

## 5. Reliability

| Atributo | Implementación |
|----------|----------------|
| **Maturity** | Manejo de errores con `try/catch` en services. HttpException con códigos HTTP apropiados. |
| **Availability** | Health check endpoint (`GET /`). DB connection pool con reconexión. |
| **Fault tolerance** | Offline-first en mobile: caché SQLite local con sincronización diferida (`syncPendingReportes`). |
| **Recoverability** | Historial de cambios en `reporte_historial_estado`. Migraciones Drizzle versionadas. |

## 6. Security

| Atributo | Implementación |
|----------|----------------|
| **Confidentiality** | JWT con expiración. Passwords hasheados con bcrypt. HTTPS en producción. |
| **Integrity** | Validación de roles (AGRICULTOR/MODERADOR) en cada endpoint. Guards JWT + Roles. |
| **Non-repudiation** | Auditoría de cambios de estado con `usuarioId` y `fechaCambio`. |
| **Accountability** | Cada acción registrada con usuario autenticado. Tokens de push asociados a usuario. |

## 7. Maintainability

| Atributo | Implementación |
|----------|----------------|
| **Modularity** | Módulos NestJS con dominio separado. Cada feature tiene su propio controller, service, repository, DTOs. |
| **Analyzability** | Logging estructurado. `PerformanceMonitor` registra tiempos lentos. Código tipado con TypeScript strict. |
| **Modifiability** | Drizzle ORM schema centralizado. Migraciones automáticas. Inyección de dependencias NestJS. |
| **Testability** | Jest configurado. Tests unitarios por módulo. Mocks para DB y servicios externos. |
| **Reusability** | DbModule @Global. PushService @Global. Guards y decorators compartidos en `common/`. |

## 8. Portability

| Atributo | Implementación |
|----------|----------------|
| **Adaptability** | Backend multiplataforma (Node.js). Mobile Expo (iOS/Android). Configuración vía `.env`. |
| **Installability** | Docker Compose para infraestructura. `npm install` + `npm run db:migrate` para setup. |
| **Replaceability** | API REST permite reemplazar frontend sin cambios. MinIO es S3-compatible (migrable a AWS S3). |

---

## Quality Gates

Antes de cada commit:

```bash
npm run build        # TypeScript compilation
npm run lint         # ESLint
npm run test         # Jest unit tests
npm run db:generate  # Drizzle migrations actualizadas
```

## Monitoreo

- `PerformanceMonitor` loguea endpoints con tiempo > 1000ms
- Errores no capturados se registran con stack trace
- Endpoint `GET /health` para monitoreo de disponibilidad
